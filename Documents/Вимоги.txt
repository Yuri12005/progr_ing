1. Функціональні вимоги

Опис вікон програми
1.1.Вхід/Реєстрація користувача
-Елементи інтерфейсу:
  - Поле для введення електронної пошти.
  - Поле для введення паролю.
  - Кнопка "Увійти" та "Зареєструватися".
- Функціональність:
  - Користувач вводить дані для входу або реєстрації.
  - При успішній авторизації користувач переходить до головного меню.
  - Реєстрація зберігає дані користувача у базі даних.

1.2. Головне меню
- Елементи інтерфейсу:
  - Кнопка "Мої картки".
  - Кнопка "Тести".
  - Кнопка "Рейтинг".
  - Кнопка "Налаштування".
  - Відображення поточного рангу користувача (наприклад, "Початківець").
- Функціональність:
  - Кожна кнопка веде до відповідного вікна.
  - Відображається прогрес користувача (балів, завершених тестів).

 1.3. Мої картки
- Елементи інтерфейсу:
  - Список доступних карток (назва, дата створення).
  - Кнопка "Додати нову картку".
  - Пошукове поле для фільтрації карток.
- Функціональність:
  - Користувач може переглядати, редагувати, видаляти картки.
  - Можливість створювати нові картки вручну:
    - Поле для введення питання.
    - Поле для введення відповіді.
  - Картки зберігаються у базі даних.

1.4. Тести
- Елементи інтерфейсу:
  - Список доступних тестів (назва, кількість питань).
  - Кнопка "Створити новий тест".
  - Кнопка "Пройти тест".
- Функціональність:
  - Тести автоматично генеруються на основі карток користувача, оскільки кожна картка містить питання та відповідь.
  - Також можна завантажити текстовий файл (наприклад, txt), і програма за допомогою OpenAI API згенерує тест з питань та відповідей.
  - Після проходження тесту відображається результат (кількість правильних відповідей, бали).
  - Бали нараховуються за кожен тест.

1.5. Рейтинг
- Елементи інтерфейсу:
  - Таблиця лідерів (ім'я користувача, бали, ранг).
  - Відображення поточного місця користувача.
- Функціональність:
  - Користувач може переглядати свій рейтинг та порівнювати його з іншими.

1.6. Налаштування
- Елементи інтерфейсу:
  - Поле для зміни паролю.
  - Кнопка "Вихід з акаунту".
- Функціональність:
  - Користувач може змінювати особисті дані та виходити з системи.

Перехід між вікнами
- Перехід між вікнами здійснюється через кнопки у головному меню.
- Для модальних вікон (наприклад, "Додати картку") використовується popup-екран.

Типові користувачі
- У програмі немає ролей. Всі користувачі мають однаковий набір функцій.



2. Нефункціональні вимоги

Мова інтерфейсу
- Інтерфейс програми підтримує лише українську мову.

Системні вимоги
- Операційна система: Windows 10/11.
- Процесор: Мінімум 2 ГГц.
- Оперативна пам’ять: Мінімум 4 ГБ.
- Місце на диску: Мінімум 500 МБ.

База даних
-  тип бази даних: SQL Server (зовнішня).
- Дані для зберігання:
  - Профілі користувачів (ім'я, електронна пошта, пароль, бали, ранг).
  - Картки (питання, відповідь, дата створення).
  - Результати тестів (кількість правильних відповідей, дата проходження).

Вимоги до безпеки
- Авторизація:
  - Паролі користувачів зберігаються у зашифрованому вигляді
  - Використання HTTPS для передачі даних.
- Конфіденційність:
  - Дані користувачів (профіль, картки, результати тестів) доступні лише після авторизації.


Гейміфікація
- Нарахування балів:
  - За кожну правильну відповідь у тесті: +10 балів.
  - За повне проходження тесту (100% правильних відповідей): додатково +20 балів.
  - Бали накопичуються та відображаються у профілі користувача.
- Ранги:
  - Початківець (0–100 балів).
  - Любитель (101–500 балів).
  - Професіонал (501+ балів).

Інтеграція з OpenAI
- Використовується OpenAI API для генерації тестів з тексту, який надається у файлі:
  - Користувач завантажує текстовий файл.
  - Програма аналізує текст за допомогою OpenAI API та генерує питання та відповіді.
  - Згенеровані питання та відповіді зберігаються у форматі карток.
- Ліміти запитів враховуються у налаштуваннях сервера.
